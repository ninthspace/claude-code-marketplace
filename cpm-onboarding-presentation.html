Getting Started with CPM

A practical guide to structured planning in Claude Code.
What it is, how it works, and when to use it.

Overview

What is CPM?

  • A planning plugin for Claude Code — adds slash commands called "skills"
  • Each skill handles a different planning stage through guided conversation
  • Produces written markdown artifacts — committed to your repo, readable by anyone
  • Conversational, not bureaucratic — asks questions one at a time, builds on your answers
  • Every step is optional — match planning depth to task complexity

Decision Guide

When to use what

⚡ Quick fix

Typo, one-liner, small bug. Don't use CPM. Just do it, or use Claude Code's native /plan.

🎯 Known scope

Feature is clear, just need structure. Jump to /cpm:spec or /cpm:epics.

🤔 Unclear requirements

Not sure what to build or why. Start at /cpm:discover to understand the problem first.

🚀 Major initiative

New product, big feature, cross-cutting change. Use the full pipeline from discover through do.

Tips

Getting the most out of CPM

  • 🎯
    Start anywhere. The pipeline is a suggestion, not a requirement. Jump to whatever skill matches where you are.
  • 📁
    Pass file paths. Most skills accept input: /cpm:spec docs/briefs/01-brief-auth.md. No path? Claude auto-discovers available artifacts.
  • 🔄
    Use pivot, not start over. /cpm:pivot amends existing artifacts and cascades changes — no need to re-run the full pipeline.
  • 📚
    Feed the library. Import coding standards, architecture docs, API specs with /cpm:library — skills reference them automatically during planning and execution.
  • ✅
    Commit your docs. Planning artifacts in the repo mean the whole team sees what was decided, why, and what's left to do.
